# PostFlow - Блог-платформа с API

PostFlow - это современная блог-платформа с REST API, разработанная на Django и Django REST Framework. Приложение позволяет пользователям создавать посты, оставлять комментарии и управлять своим контентом.

## ✨ Возможности

### Пользователи
- Регистрация и аутентификация
- Просмотр и редактирование профиля
- Разграничение прав доступа (администратор/пользователь)

### Посты
- Создание, чтение, обновление и удаление постов
- Поддержка изображений
- Автоматическая проверка возраста автора (18+)
- Фильтрация запрещенных слов в заголовке

### Комментарии
- Комментирование постов
- Редактирование и удаление своих комментариев
- Фильтрация комментариев по посту

### Безопасность
- Валидация пароля (мин. 8 символов, наличие цифр)
- Валидация email (только mail.ru и yandex.ru)
- Права доступа на основе ролей

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```
git clone https://github.com/morozoVik/postflow.git
cd postflow
```

### 2. Создание виртуального окружения

```
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/Mac
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

```
pip install -r requirements.txt
```

### 4. Настройка базы данных PostgreSQL

Создайте базу данных и пользователя:

```
CREATE DATABASE postflow_db;
CREATE USER postflow_user WITH PASSWORD 'your_password';
ALTER ROLE postflow_user SET client_encoding TO 'utf8';
ALTER ROLE postflow_user SET default_transaction_isolation TO 'read committed';
ALTER ROLE postflow_user SET timezone TO 'UTC';
GRANT ALL PRIVILEGES ON DATABASE postflow_db TO postflow_user;
```

### 5. Настройка переменных окружения

Создайте файл .env в корне проекта:

```
DEBUG=True
SECRET_KEY=your-secret-key-here
DB_NAME=postflow_db
DB_USER=postflow_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
```

### 6. Применение миграций

```
python manage.py createsuperuser
```

### 8. Запуск сервера разработки

```
python manage.py runserver
```

Приложение будет доступно по адресу: http://127.0.0.1:8000/

## 📡 API Эндпоинты

API построен на Django REST Framework и использует стандартные HTTP методы. Все эндпоинты доступны по базовому URL: `http://127.0.0.1:8000/api/`

| Поле | Тип | Обязательное | Описание | Валидация |
|------|-----|
### 🔐 Аутентификация

API поддерживает два способа аутентификации:
- **Session Authentication** - используется для браузерного API
- **Basic Authentication** - для тестирования через командную строку

Для доступа к защищенным эндпоинтам необходимо авторизоваться через `/api-auth/login/` или использовать заголовок `Authorization`.

---

### 👤 Пользователи (/users/)

|Метод | URL | Описание | Доступ |
|------|-----|-----|-----|
|POST | /users/ | Регистрация | Все |
|GET	|/users/	|Список пользователей	|Только админ|
|GET	|/users/{id}/	|Просмотр профиля	|Админ/Владелец|
|PATCH	|/users/{id}/	|Обновление профиля	|Админ/Владелец|
|DELETE	|/users/{id}/	|Удаление	|Только админ|

**Регистрация:**

```
POST /api/users/
{
    "username": "john",
    "email": "john@mail.ru",
    "password": "Pass123",
    "birth_date": "1995-05-15"
}
```

### 📝 Посты (/posts/)

|Метод	|URL	|Описание	|Доступ|
|---|---|---|---|
|GET	|/posts/	|Список постов	|Все
|POST	|/posts/	|Создание поста	|Авторизованные
|GET	|/posts/{id}/	|Просмотр поста	|Все
|PATCH	|/posts/{id}/	|Обновление поста	|Автор/Админ
|DELETE	|/posts/{id}/	|Удаление поста	|Автор/Админ

**Создание поста:**

```
POST /api/posts/
{
    "title": "Мой пост",
    "content": "Текст поста"
}
```

### 💬 Комментарии (/comments/)

|Метод	| URL	                  |Описание	|Доступ|
|---|-----------------------|---|---|
|GET	| /comments/            |	Список комментариев	|Все
|GET	| /comments/?post_id=1	 |Комментарии к посту	|Все
|POST	| /comments/	           |Создание комментария	|Авторизованные
|PATCH	| /comments/{id}/	      |Обновление комментария	|Автор/Админ
DELETE	| /comments/{id}/	      | Удаление комментария	 |Автор/Админ

**Создание комментария:**

```
POST /api/comments/
{
    "post": 1,
    "content": "Отличный пост!"
}
```

### ✅ Валидация

- Пароль: минимум 8 символов, обязательны цифры
- Email: только mail.ru или yandex.ru
- Возраст автора: старше 18 лет
- Заголовок поста: без слов "ерунда", "глупость", "чепуха"
